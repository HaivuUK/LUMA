Thank you for your interest in contributing to LUMA! We welcome contributions from everyone. 
Your contributions help us improve the website and make it more useful for the community.

### How To Help
#### 1. Feature Requests
Share ideas for new features or improvements by [opening a GitHub Issue](https://github.com/HaivuUK/luma/issues) or check our [Roadmap](../../roadmap/) to see what's planned.

#### 2. Bug Reports
Found a bug or security issue? Report it directly in our [Security Center](https://github.com/HaivuUK/luma/security)!

#### 3. Improve Code
Set up LUMA on your computer, run it locally, and make changes using this [Build from Source Guide](../../docs/development/building_from_source).

#### 4. Translations
Help translate LUMA documentation into another language, with our [Translation Guide](../translate).

#### 5. Spreading The Word
Share LUMA on social media or with friends and family.

### Key areas we are looking for help with
- **Performance improvements**: Profiling and optimisation patches. More efficient code is nice, and I am bad at writing it.
- **Additional element types**: Support for more FE element types
- **Extended file formats**: New mesh and CT Image formats remove restrictions around workflows.
- **Test datasets**: Varied mesh and image samples help us ensure LUMA works for everyone and remove edge cases.
- **Documentation**: Usage examples and tutorials for new features, as well as translations (see [Translation Guide](../translate)).
